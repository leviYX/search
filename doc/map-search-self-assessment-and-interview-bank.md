# 地图搜索自测题库与面试题

更新时间：2026-03-29

这份题库分两种用途：

- 自测：检查你是否真的掌握了地图搜索核心能力
- 面试：如果你以后想面高级搜索 / 本地搜索岗位，这些问题基本绕不开

## 1. 信息检索基础

### 题目 1

什么是倒排索引？它为什么适合搜索系统？

答案要点：

- term 到 doc list 的映射
- 适合关键词检索
- 能高效做交并集和打分

### 题目 2

BM25 和 TF-IDF 的关键差异是什么？

答案要点：

- BM25 对词频收益有饱和处理
- BM25 有文档长度归一化
- BM25 更适合现代检索场景

### 题目 3

Recall 和 NDCG 分别适合衡量什么？

答案要点：

- Recall 更关注有没有召回
- NDCG 更关注排序位置和相关性等级

## 2. 搜索引擎与工程基础

### 题目 4

Lucene 的 segment、refresh、commit、merge 分别是什么？

答案要点：

- segment 是索引基本物理单元
- refresh 让新数据可搜
- commit 保证持久化
- merge 减少 segment 数量并优化查询性能

### 题目 5

为什么 explain 很重要？什么时候只看 explain 还不够？

答案要点：

- explain 能帮助你定位分数来源
- explain 只看单条 case，不足以说明整体收益
- 必须结合评估和 query 分桶

## 3. Query 理解

### 题目 6

“南站”为什么是典型的地图搜索难题？

答案要点：

- 歧义强
- 强依赖城市上下文和用户位置
- 可能对应交通枢纽、道路、商圈等不同实体

### 题目 7

“北京朝阳大悦城星巴克”这种 query 该怎么拆？

答案要点：

- 城市：北京
- 区域：朝阳
- 地点限定：大悦城
- 品牌实体：星巴克
- 应优先理解为地点约束下的品牌搜索

### 题目 8

地址解析失败和地址召回失败有什么区别？

答案要点：

- 解析失败是结构化理解出错
- 召回失败是结构化后没找到候选
- 两者修复路径不同

## 4. 地图搜索核心能力

### 题目 9

为什么地图搜索不能只按文本分排序？

答案要点：

- 地图搜索要同时考虑文本、距离、区域、场景
- 最近的不一定最好，文本最像的也不一定最好

### 题目 10

附近搜和 POI 搜、地址搜最大的区别是什么？

答案要点：

- 附近搜更依赖用户位置和距离
- 地址搜更依赖结构化解析和 geocoder
- POI 搜更偏实体匹配和别名召回

### 题目 11

H3 / S2 这种空间索引在地图搜索里解决什么问题？

答案要点：

- 提高地理邻近检索和空间聚合效率
- 更好表达空间层级和覆盖关系

## 5. 评估与 badcase

### 题目 12

为什么只看 CTR 会误导搜索优化？

答案要点：

- CTR 受展示位置和流量结构影响很大
- 无法区分召回问题和排序问题
- 不能替代离线评估与 badcase 分析

### 题目 13

如何构建地图搜索的 query 分桶？

答案要点：

- 按品牌词、地址词、地标词、类目词、周边词、歧义词等分桶
- 分桶是为了知道不同策略影响了什么场景

### 题目 14

一个 badcase 到底该归因到召回还是排序，怎么判断？

答案要点：

- 正确结果根本没进候选集，优先看召回
- 正确结果进了候选但排位低，优先看排序
- 还要排除 Query 理解和数据质量问题

## 6. 排序、LTR 与向量

### 题目 15

LTR 为什么不能替代特征设计？

答案要点：

- 模型只能利用已有特征
- 特征质量决定排序上限
- 样本偏差也会直接影响模型表现

### 题目 16

向量检索在地图搜索里最适合放在哪些位置？

答案要点：

- 长尾补召回
- 别名或语义相似召回
- TopK rerank

### 题目 17

为什么品牌词、地址词通常不适合让向量召回主导？

答案要点：

- 这类 query 对精确实体和结构化匹配要求高
- 语义相似未必等于地理场景正确

## 7. 架构与负责人视角

### 题目 18

如果效果更好，但 P99 延迟上升 40%，你怎么判断是否值得？

答案要点：

- 先看收益是否覆盖成本
- 看是否能只在特定 query 分桶启用
- 看是否能通过二阶段或缓存降低开销

### 题目 19

如果线上实验整体变好，但地址词分桶变差，你会怎么处理？

答案要点：

- 不能只看整体
- 要分析是否流量大类掩盖了关键场景退化
- 必要时 query 路由或分场景策略

### 题目 20

资深地图搜索工程师和普通搜索工程师的差异在哪里？

答案要点：

- 不只是写策略，而是能构建评估闭环、抽象方法论、做架构权衡、主导专项

## 8. 负责人级开放题

### 题目 21

请设计一个“附近咖啡店”搜索链路，包括召回、排序、评估和风险控制。

回答要点：

- 召回：类目召回 + geo 过滤
- 排序：文本、距离、热度、营业状态、质量分
- 评估：分桶 NDCG、零结果率、首条满意率
- 风险：距离过强导致低质量结果靠前，热度过强导致“附近”感变弱

### 题目 22

请设计一个地址搜索专项治理方案。

回答要点：

- 问题拆分：parser、召回、排序、数据
- 指标：Top1 准确率、地址 query 成功率、零结果率
- 实验：按地址类型分桶评估
- 风险：不同城市地址标准差异大

### 题目 23

请评估是否要引入向量检索到现有地图搜索主链路。

回答要点：

- 先明确定义目标 query
- 验证资源与延迟成本
- 比较 lexical、vector、hybrid 在不同 query 分桶的效果
- 不要默认“全量主链路替换”

## 9. 90 天自测清单

如果你完成了前 90 天，应该能回答下面这些问题：

- 我能否独立搭建最小搜索系统
- 我是否真的理解 BM25 和 explain
- 我是否有一套可用的 badcase 集
- 我是否建立了最小离线评估集
- 我是否能区分品牌词、地址词、附近词的策略差异
- 我是否知道自己当前最大短板是什么

## 10. 6 个月自测清单

- 我能否独立设计 geocoder 或地址搜索实验
- 我是否能稳定做 query 分桶和 badcase 归因
- 我是否已经有一套排序特征库
- 我是否能判断向量检索适不适合某类 query
- 我是否能写出完整实验报告

## 11. 9 个月自测清单

- 我能否设计地图搜索的多阶段链路
- 我能否主持一个搜索效果专项
- 我能否平衡效果、延迟、稳定性和资源成本
- 我能否把经验抽象为团队规范和模板
- 我能否独立完成一份负责人级技术方案

## 12. 推荐使用方式

- 每周做一次小测，检查是否真的理解
- 每个月抽 10 道题写答案
- 每个阶段结束时，至少写 1 道开放题的完整方案

如果有题你答不上来，不要只补答案，要回到 badcase、实验和资料里补链路理解。
