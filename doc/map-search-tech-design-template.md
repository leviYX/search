# 地图搜索技术方案模板

更新时间：2026-03-29

这份模板面向你开始以“负责人”视角推进专项时使用。重点不是把文档写长，而是把问题拆清楚、证据补齐、风险说透。

## 1. 技术方案标准结构

### 1.1 背景

- 业务背景是什么
- 当前问题是什么
- 问题是否长期存在，影响多大
- 为什么现在要解决

### 1.2 目标

目标必须具体，不要写成“提升搜索体验”这种空话。

建议写法：

- 地址词 Top1 准确率提升 `x%`
- “附近类目词” NDCG@5 提升 `x%`
- 零结果率下降 `x%`
- P99 延迟不超过 `x ms`

### 1.3 问题拆解

把问题拆到链路级别。

示例：

- Query 理解问题
- 候选召回问题
- 排序问题
- 地理约束问题
- POI 数据质量问题
- 在线性能问题

### 1.4 基线分析

至少要有：

- 指标基线
- badcase 样本
- query 分桶表现
- 当前链路图

### 1.5 方案设计

方案至少分成 4 个层次：

- 输入和依赖
- 核心策略
- 评估方案
- 上线方案

### 1.6 风险与回滚

必须明确：

- 哪些 query 可能退化
- 哪些系统指标可能恶化
- 如何灰度
- 如何监控
- 如何回滚

## 3. 常见专项的方案写法

### 3.1 地址搜索专项

问题拆解建议：

- parser 是否稳定
- 地址槽位是否足够
- 候选召回是否覆盖
- 排序是否能识别地址完整度与区域一致性

指标建议：

- 地址 query Top1 准确率
- 地址 query 零结果率
- 地址 query 成功率

### 3.2 附近搜索专项

问题拆解建议：

- 用户位置是否准确传入
- 搜索半径是否合理
- 距离与热度是否平衡
- 营业状态是否参与排序

指标建议：

- nearby query NDCG@5
- 首条满意率
- 人均点击距离

### 3.3 品牌词专项

问题拆解建议：

- 品牌别名是否完整
- 品牌主实体是否靠前
- 分店和品牌馆聚合是否合理

指标建议：

- 品牌词 Top1 准确率
- 品牌词主实体首位率

## 6. 评审时必须回答的问题

1. 这个方案解决的是哪个链路问题
2. 证据是否充分
3. 离线收益是否可信
4. 哪些 query 可能退化
5. 延迟和资源代价是否能接受
6. 如果效果不符合预期，回滚是否简单

## 7. 负责人最容易犯的错误

- 目标不清晰
- 问题拆解不到链路层
- 没有 query 分桶
- 只描述方案，不描述风险
- 不提前定义回滚条件

## 8. 最后的建议

好方案不是“看起来先进”，而是：

- 目标清楚
- 证据充足
- 方案可执行
- 风险可控
- 收益可验证
